Introdução


1. Introdução

O projeto de configuração de servidores utiliza o sistema operacional Linux Debian e também um ambiente que simula o acesso do cliente ao servidor. O projeto realizado no sistema de gerenciamento de máquinas virtual Virtual-Box, consiste em mostrar, passo a passo, todo o processo de configuração de ambos os ambientes.

A máquina servidor, identificada como ‘serverProj’, terá as configurações de servidores DHCP, DNS, Web e NFS, a fim de permitir que a máquina cliente, identificada como ‘clienteProj’, receba os parâmetros adequados de configuração e se comunique ao servidor, possibilitando assim a transferência de arquivos.

O range de IPs definido pelo servidor DHCP é 192.168.1.5 - 192.168.1.10. O servidor serverProj terá IPv4 192.168.1.1 e o cliente clienteProj receberá o IPv4 192.168.1.1

O servidor Web será o Apache, servidor Web de código aberto, para disponibilizar o acesso a páginas mantidas no ambiente serverProj. No projeto em questão.

As configurações dos serviços de compartilhamento de arquivo foram usados o protocolo NFS, Network File System, onde criamos um diretório chamado cnfs, desta forma, fizemos as configurações na máquina cliente e acessar os diretórios arquivos do servidor.

Servidores e Sistemas Operacionais


2. Servidores e Sistemas Operacionais


2.1 Servidores

Os Servidores de computação (servers) são computadores utilizados para entregar serviços para as demais máquinas de uma rede, como a Internet. Geralmente, possuem Hardware específico e Sistemas Operacionais próprios. Esses serviços podem oferecer serviços de naturezas distintas, como por exemplo, arquivos, correio eletrônico ou nomes de domínio.

O conjunto desses sistemas e hardware, também definidos como arquitetura, é chamada de modelo cliente-servidor. É utilizada em redes de pequeno, médio e grande porte e realiza o armazenamento e compartilhamento de dados entre clientes.

O termo servidor é amplamente aplicado a computadores completos, embora um servidor possa ser um software, partes de um sistema computacional ou até mesmo máquinas que não são, necessariamente, um computador (IoT).

Os servidores podem fornecer diversas funcionalidades, muitas vezes chamadas de "serviços", como a partilha de dados ou de recursos do sistema entre vários clientes. Um único servidor pode servir vários clientes, e um único cliente pode usar vários servidores. Um processo cliente pode ser executado no mesmo dispositivo ou pode se conectar através de uma rede para um servidor em um dispositivo diferente.


2.2 Sistemas Operacionais

Sistema Operacional é um software ou conjunto de softwares cuja função é administrar e gerenciar os recursos de um sistema, desde seus componentes de hardware aos sistemas de arquivos e programas de terceiros, estabelecendo a interface entre o computador e o usuário.

Entenda como um “computador” qualquer máquina de processamento automático de dados, como um desktop ou notebook, um celular ou tablet, um smartwatch, um servidor, um videogame, um set-top box, um roteador ou outro quaisquer outros aparelhos.

O sistema operacional introduz uma “camada de abstração” entre o hardware e o usuário, que transforma comandos no mouse ou teclado e solicitações do sistema, como gerenciamento de recursos, CPU, Memória RAM, em linguagem de máquina, enviando instruções ao processador. Este último os traduz para código binário, executa os comandos e envia as respostas como informações que aparecem na sua tela. Um sistema operacional contém componentes divididos entre os para o usuário, como bibliotecas, programas e interface, e as instruções que compõem o seu núcleo, kernel.

No projeto em questão, será abordado o sistema operacional Linux Debian, feita toda as configurações abordadas.

Servidores DHCP


4. Servidores DHCP


Dynamic Host Configuration Protocol ou simplesmente DHCP, é o nome de um protocolo TCP/IP que oferece serviços de configuração dinâmica em redes. Este protocolo fornece concessão de endereços IP de host, máscara de sub-rede e default gateway.

Para o projeto, instalamos o arquivo de configuração do servidor DHCP a fim de fornecer estes serviços para o cliente e, portanto, ambas as máquinas se comunicarem.


4.1 Instalando servidor DHCP no serverProj

Neste passo, estará todo o processo de instalação do servidor DHCP e configuração da interface das máquinas.


1º Passo: Inserir o comando apt-get install isc-dhcp-server.


2º Passo: Criando escopo de rede do servidor.

Este processo consiste em copiar e renomear o arquivo dhcpd.conf para dhcpd.conf.old. Após a cópia, remover o arquivo anterior dhcpd.conf. Desta forma existira um arquivo onde não será feita alterações.


3º Passo: Criando outro arquivo dhcpd.conf.

Onde será feita a configuração do espaço de rede. Comando, nano /etc/dhcp/dhcpd.conf.


4º Passo: Adicionando a interface no servidor.

Comando nano/etc/default/isc-dhcp-server.


5º Passo: Configurando a interface adicionada.

Comando nano /etc/network/interfaces.


6º Passo: Verificar se as configurações estão corretas.

Para isso, o comando dhcpd -t mostrará se alguma configuração esteja errada.

No nosso exemplo, tudo correto.


7º Passo: Reiniciar a interface e a máquina servidor.

Para este processo será necessário inserir o comando ifdown enp0s8. Após este processo ifup enp0s8.

OBS: Em algumas máquinas será necessário reiniciá-las e também, adicionar uma outra interface nas configurações de rede no Virtual-Box. Passos; Ir em Configurações, Redes, Adaptador 2, habilitar nova interface e deixar como rede Interna. Neste mesmo local, posteriormente foi modificada para rede NAT, assim, o servidor se conectará com a internet.


8º Passo: Verificando a o Servidor DHCP ativo.

Inserir o comando systemctl status isc-dhcp-server.


9º Passo: Verificando as interfaces e com os devidos IPs.

Com o comando ip -a addr mostrara as interfaces.


4.2 Configurando DHCP no cliente


Neste processo é importante mudar as configurações padrões de rede no Virtual-Box da máquina cliente. Para isso, nas configurações de rede, trocar de NAT para rede interna no adaptador de rede. Após este processo, a máquina cliente se conectará com o servidor.


1º Passo: Verificando novo endereço de IP do cliente.

Comando ip -a addr.


2º Passo: Testando a conexão com o comando ping 192.168.1.1.

Servidores DNS


5. Servidores DNS


O Sistema de Nomes de Domínio, é um sistema hierárquico e distribuído de gestão de nomes para computadores, serviços ou qualquer máquina conectada internet.

O DNS apresenta uma arquitetura cliente/servidor, podendo envolver vários servidores DNS na resposta a uma consulta. O servidor DNS resolve nomes para os endereços IP e de endereços IP para os nomes respectivos, permitindo a localização de hosts num determinado domínio.


5.1 Instalando e configurando servidor DNS


Para iniciar a instalação do servidor DNS, primeiro terá que baixar os pacotes essenciais, desta forma, fazer as devidas configurações nos arquivos.

Sendo assim, executar o comando apt install bind9 dnsutils. Observe que também estará baixando pacotes extras chamado dnsutils. Pois, este pacote serve para fazer consultas nos servidores DNS para descobrir informações sobre hosts da Internet.

Configurar as zonas de DNS. Uma zona de DNS é um espaço administrativo que permite um controle mais detalhado dos componentes do DNS, como os nameservers autoritativos. A zona de DNS começa em um domínio dentro da árvore e também pode se estender por subdomínios, de forma que múltiplos subdomínios possam ser gerenciados por uma entidade.


1º Passo: Executar o comando nano /etc/bind/named.conf.


2º Passo: Criando arquivos de pesquisa de zona direta, o mesmo vinculado ao arquivo named.conf.

Usar o comando nano /etc/bind/projeto.zone.


3º passo: Criando outro arquivo para interface de loopback.

Ira estar localizado no mesmo caminho, /etc/bind/. Portanto, usar o comando nano /etc/bind/projeto.rev.


4º Passo: Inserir o IP do servidor no arquivo resolv.conf.

Pois o mesmo é responsável por buscar o servidor DNS para resolver nomes na rede.

Comando nano /etc/resolv.conf.


5º Passo: Verificar se todas as configurações estão corretas.

Usar os seguintes comandos:

  • named-checkconf /etc/bind/named.conf
  • named-checkzone projeto.br /etc/bind/projeto.zone
  • named-checkzone 168.192.in-addr.arpa

    • 6º Passo: Reiniciar o serviço Bind9.

    Usar o comando systemctl restart bind9. Após este processo, verificar o status do servidor DNS. Comando systemctl status bind9.


    7º Passo: O arquivo hosts tem a função de mapear um nome para um endereço IP.

    Normalmente, é utilizado em conjunto com uma Virtual Host para configurar um ambiente de desenvolvimento. Comando nano /etc/hosts.


    8º Passo: Testes com o dig, é uma ferramenta de redes de computadores, utilizada para consultas sobre registos de DNS de um domínio, host ou IP.

    Comando dig 192.168.1.1.


    9º Passo: Usando dig, Domain Information Groper, para resolver um nome.

    Comando dig projeto.br.


    10º Passo: Teste com nslookup.

    Comando nslookup 192.168.1.1 e nslookup ns.projeto.br.


    11º Passo: Verificar interfaces de rede e postas.

    Comando netstat -anu|grep 53.


    5.2 Testando configuração com máquina cliente


    Este processo consiste em indicar na máquina cliente o servidor DNS. Também no arquivo resolv.conf, inserir o domínio e endereço de servidor DNS.


    1º Passo: Inserir os domínio e endereço.

    Comando nano /etc/resolv.conf.


    2º Passo: Testes com ferramenta dnsultis previamente instalada.

    Para usá-la, usar o comando dig mais o endereço ou nome do domínio e, nslookup mais endereço ou nome do domínio. Teste usando dig 192.168.1.1.


    3º Passo: Teste com nslookup.

    Comandos nslookup vitor.projeto.br, nslookup jonas.projeto.br, nslookup jcarlos.projeto.br.

Servidor WEB


6. Servidor WEB


Servidores WEB são computadores com software específicos responsável por aceitar pedidos HTTP de clientes, geralmente os navegadores, e servi-los com respostas HTTP, incluindo opcionalmente dados, que geralmente são páginas web, tais como documentos HTML com objetos embutidos, imagens, textos, mídia e etc. Neste projeto, foi instalado o servidor WEB Apache.


6.1 Instalando e configurando servidor WEB


O processo de instalação consiste em usar as ferramentas e pacotes do apache2. Para instalar, com o comando apt-get install apache2 foi baixado todos os serviços necessários para o projeto.

A princípio, será necessário ativar os modulo de segurança SSL. Ao ativar este serviço os servidores podem enviar tráfego com segurança entre servidores e clientes sem a possibilidade de mensagens serem interceptadas por agentes externos.


1º Passo: Para ativar o serviço SS.

Comando a2enmod ssl. Após este processo, listamos os serviços ssl.conf e ssl.load. Diretório /etc/apache2/.


2º Passo: Reiniciar o serviço Apache2.

Comandos systemctl restart apache2.service e para ver o resultado, systemctl status apache2.service.


3º Passo: Criando um certificado SSL com o arquivo default-ssl.conf.

Este certificado é usado para criptografar o conteúdo enviado para clientes. Comando a2ensitedefault-ssl.conf. Após este processo, reiniciamos o serviço. Diretório /var/www/html/.


4º Passo: Fazer uma cópia do arquivo 000-default.conf editá-lo.

No projeto em questão utilizamos o nome meuprojeto. Processo feito no diretório /etc/apache2/.


5º Passo: Altera arquivo novo chamado meuprojeto.

No item ServerName adicionar o nome do site. No projeto usamos o site www.meuprojeto.com. No item DocumentRoot, adicionar o caminho onde está localizado seus arquivos html e etc.


6º Passo: Ativar e reiniciar o arquivo meuprojeto e os serviços Apache2.

Comando a2ensite meuprojeto.conf. Diretório /etc/apache2/sites-available.


6.2 Testes na máquina cliente


O teste na máquina cliente consiste em usar um arquivo previamente criado no servidor WEB, localizado no diretório var/www/html onde consta outros arquivos.


1º Passo: Testando acesso a página já com o certificado SSL.


2º Passo: Configuração para testar o DNS resolver nome localmente.

Observação: Para realizar este processo, será necessário altera o arquivo resolv.conf e desabilitar nameservar com o endereço 192.168.1.1. Após este processo, acessar o arquivo /etc/host/ e adicionar o endereço 192.168.1.1 e o nome do site. Desta forma, basta digitar no navegador da máquina cliente www.meuprojeto.com que o site irá abrir.

Servidor de Compartilhamento de Arquivo com Protocolo NFS


7. Servidor de Compartilhamento de Arquivo com Protocolo NFS


O protocolo NFS, Network File System, é um sistema de arquivos distribuídos responsável por compartilhar diretórios e arquivos entre servidores e clientes. O cliente NFS tem por finalidade tornar o acesso remoto transparente para o usuário do computador, e esta interface cliente e servidor, executada pelo NFS através dos protocolos Cliente-Servidor, fica bem definida quando o usuário, ao chamar um arquivo/diretório no servidor, lhe parece estar acessando localmente, sendo que está trabalhando com arquivos remotamente.


7.1 Instalando Servidor NFS


O processo de instalação do servidor consiste em baixar o pacote do protocolo NFS com o comando apt-get install nfs-kernel-server. Após isso, criar um diretório para poder fazer o compartilhamento entre o cliente, sendo assim, concluir sua montagem no diretório /etc/mnt.


1º Passo: Acessar o diretório /etc/mnt/ e criar uma parta, no projeto criamos o cnfs e projetoNFS.


2º Passo: Posteriormente, criamos um arquivo chamado projeto.txt e adicionamos informações com o comando nano projeto.txt.


3º Passo: Neste passo é editar o arquivo /etc/exports.

Nele deverão ser colocados o diretório do servidor que vai ser compartilhado com a máquina cliente, o número do IP da máquina que estará acessando o servidor e por fim as opções de segurança. Comando nano /etc/exports, adicionar o diretório cnfs criado anteriormente e IP 192.168.1.0/24, por fim, as permissões de acesso.


4ª Passo: Neste processo, inserimos o comando exportfs -a é usado para manter a tabela dos sistemas de arquivos exportados via NFS.

Essa tabela é armazenada no arquivo /var/lib/nfs/etab que é lido pelo daemon mountd quando um cliente solicita acesso para montar o sistema de arquivos exportado.


5º Passo: Após concluir a instalação, reiniciar o sistema NFS.

Comando systemctl restart nfs-server e comando para ver o status, systemctl status nfs-server.


7.2 Instalando NFS na máquina cliente


Na máquina cliente, temos que fazer a montagem do diretório compartilhado pelo servidor. Os parâmetros utilizados é comando mount, tipo do protocolo, IP do servidor e endereço de onde está sendo montado o compartilhamento na máquina cliente.


1º Passo: Desta forma, inserimos o comando mount -t nfs 192.168.1.1/mnt/cnfs /mnt.

Após este processo, inserir o comando mount novamente, logo, exibirá os diretórios montados com o endereço do servidor.


2º Passo: Nesta etapa, é só descer até as últimas informações do mount e observar se o mesmo foi configurado corretamente.


3º Passo: Neste processo, vamos listar no diretório /mnt as pastas com seus arquivos oriundos do servidor.

Comando cd /mnt. E por fim, podemos exibir o conteúdo do arquivo projeto.txt com o comando cat projeto.txt.
